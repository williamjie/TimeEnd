# 休息时间记录需求规划

## 需求来源

- 需求 1：结束或停止后，记录休息的时间
- 需求 2：如果再次开始记录专注，休息的时间清零

## 行为定义

| 事件 | 行为 |
|------|------|
| **专注结束**（倒计时到 0，弹出中断框） | 开始记录休息时间，界面显示「已休息 X 分钟」并实时增加 |
| **用户点击「停止」** | 同样开始记录休息时间，同上 |
| **用户点击「开始」** | 本次休息计时清零，隐藏或归零休息显示，开始新的专注 |

## 设计要点

1. **只记录“当前这段”休息**  
   不要求持久化历史休息总长；若以后要「今日休息总长」可在此基础上扩展。

2. **休息起点**  
   主进程在「专注结束」或「停止」时记录 `restStartTime = Date.now()`，并通知渲染进程。

3. **休息清零**  
   主进程在收到「开始专注」时清除 `restStartTime`，并通知渲染进程「休息清零」。

4. **界面**  
   在统计区增加一项「本次休息：X 分钟」。仅在处于休息状态时显示并每秒更新；开始专注后该项清零或隐藏。

## 实现范围

- **主进程 (main.js)**  
  - 变量：`restStartTime`（null 表示未在休息）  
  - 在计时完成回调、`stop-timer` 中设置 `restStartTime`，并 `send('rest-started', restStartTime)`  
  - 在 `start-timer` 中清除 `restStartTime`，并 `send('rest-cleared')`  
  - 可选：`get-stats` 或 `get-rest` 返回 `restStartTime`，便于窗口刷新后恢复显示  

- **渲染进程 (index.html + main.js)**  
  - 统计区增加「本次休息」区块（默认可隐藏或显示 0）  
  - 收到 `rest-started` 后保存时间戳，用 setInterval 每秒更新「已休息 X 分钟」  
  - 收到 `rest-cleared` 或开始专注后清除定时器，休息显示归零/隐藏  

- **数据**  
  - 当前不在 store 中持久化休息时长；仅内存中记录，满足「结束/停止后记录、开始后清零」即可。

## 验收

- 专注结束或点击停止后，界面出现「本次休息」并随时间增加。  
- 点击「开始」后，休息计时清零，不再增加。  
- 再次结束/停止后，休息时间从 0 重新开始计。
