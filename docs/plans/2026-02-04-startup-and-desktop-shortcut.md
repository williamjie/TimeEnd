# 开机自启动与安装时桌面快捷方式 - 需求规划

## 需求概述

1. **开机自启动**：用户可设置「开机时自动启动 TimeEnd」，关闭设置后则不再自启动。
2. **安装时桌面快捷方式**：通过安装程序安装时，在用户桌面创建 TimeEnd 的快捷方式，方便从桌面启动。

---

## 一、开机自启动

### 行为约定

| 项目 | 说明 |
|------|------|
| **开关** | 在应用内提供一处「开机自启动」开关（建议放在设置弹窗中），默认关闭。 |
| **开启** | 用户打开开关后，下次开机（或当前用户登录 Windows）时，TimeEnd 自动启动。 |
| **关闭** | 用户关闭开关后，不再加入开机自启动，下次开机不会自动启动。 |
| **持久化** | 开关状态保存到本地（如 electron-store），应用重启后保持用户选择。 |

### 技术要点（Windows）

- 使用 Electron 的 **`app.setLoginItemSettings(openAtLogin: true/false)`** 控制是否加入当前用户的「启动」项（注册表或启动文件夹）。
- 仅影响**当前用户**的登录项，不需要管理员权限。
- 应用启动时根据本地保存的开关状态调用 `setLoginItemSettings`，保证与设置一致。

### 界面建议

- 在现有「设置」弹窗中增加一行：「开机自启动」+ 复选框或开关。
- 用户勾选/打开即写入配置并调用 `openAtLogin: true`；取消勾选即 `openAtLogin: false`。

### 验收标准

1. 设置中可勾选「开机自启动」，勾选后重启电脑（或重新登录），TimeEnd 能自动启动。
2. 取消勾选后，再次开机不再自动启动。
3. 重新打开应用后，设置弹窗中的开关状态与当前是否自启动一致。

---

## 二、安装时桌面快捷方式

### 行为约定

| 项目 | 说明 |
|------|------|
| **时机** | 用户通过安装程序（NSIS）安装 TimeEnd 时。 |
| **结果** | 在用户桌面上创建名为「TimeEnd」的快捷方式，双击可启动应用。 |
| **可选** | 若 NSIS 安装向导支持「创建桌面快捷方式」选项，建议默认勾选，用户可取消。 |

### 技术要点

- 使用 **electron-builder** 的 **NSIS** 配置，在 `build.nsis` 中开启创建桌面快捷方式。
- 常见配置项：**`createDesktopShortcut`** 设为 `true`（或使用等价配置），安装完成后在桌面生成指向已安装 exe 的快捷方式。
- 快捷方式名称与 `productName` 一致（如「TimeEnd」）即可。

### 验收标准

1. 通过 `npm run build` 生成的安装包安装后，用户桌面上出现「TimeEnd」快捷方式。
2. 双击该快捷方式可正常启动 TimeEnd。

---

## 三、实施顺序建议

1. **安装时桌面快捷方式**：仅改 `package.json` 的 `build.nsis` 配置，无业务逻辑，先做。
2. **开机自启动**：在设置弹窗加开关、存配置、主进程调用 `app.setLoginItemSettings`，并应用启动时同步一次。

---

请确认以上规划是否符合预期（尤其是自启动仅针对当前用户、桌面快捷方式默认创建）。确认后再进入开发实现。
