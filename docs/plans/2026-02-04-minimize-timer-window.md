# 最小化仅显示时间并可拖动 - 需求规划

## 需求描述

用户希望增加一种「最小化」模式：在此模式下，界面**只显示正在流逝的时间**，窗口**可随意拖动**，以便在桌面有遮挡时把计时器挪到合适位置，减少遮挡、同时保持可见。

## 目标效果

| 项目 | 说明 |
|------|------|
| **显示内容** | 仅显示当前时间（如 `25:00` → `24:59` …）及可选的状态文案（如「专注中」），无统计、无配置、无按钮。 |
| **窗口形态** | 小窗口、尽量不占地方，可拖动到屏幕任意位置。 |
| **进入方式** | 在主界面提供「最小化」或类似入口（如按钮、菜单），点击后进入该模式。 |
| **退出方式** | 在最小化窗口上提供「还原」或双击等操作，恢复为完整主窗口。 |

## 行为约定

1. **进入最小化**
   - 用户点击主界面的「最小化」后，主窗口隐藏或关闭，出现一个**仅显示时间**的小窗口。
   - 小窗口内：大号时间数字 + 可选一行状态（如「专注中」/「已暂停」/「准备就绪」）。
   - 计时逻辑不变：若正在专注，时间继续走；若已暂停，显示暂停时的时间。

2. **拖动**
   - 小窗口**无标题栏**或**标题栏可拖动**，整窗可拖动，用户可随意挪到不被遮挡的位置。

3. **退出最小化（还原）**
   - 小窗口上提供明显入口（如「还原」按钮、或双击窗口）恢复主界面。
   - 还原后：主窗口重新显示，小窗口关闭；计时状态、统计数据与进入最小化前一致。

4. **强制中断**
   - 若在最小化状态下计时结束，仍按现有逻辑弹出**强制中断输入框**（可置顶、模态）；用户完成输入后，行为与当前一致。

## 技术方案（建议）

- **小窗口**：使用 Electron 新建一个**无边框、可拖动**的 `BrowserWindow`，尺寸较小（例如 180×80 或按内容自适应）。
- **内容**：单独一个极简 HTML 页面，只渲染「时间 + 状态」，通过 IPC 与主进程同步 `remaining` 和状态（专注中/已暂停/准备就绪）。
- **主窗口**：进入最小化时 `mainWindow.hide()`，还原时 `mainWindow.show()` 并 `miniWindow.close()`。
- **拖动**：无边框窗口通过 `-webkit-app-region: drag` 在页面指定区域（如整块时间区域）实现拖动；「还原」按钮区域设为 `no-drag`，保证可点击。

## 界面要点（小窗口）

- **必须**：大号时间数字（与主界面风格一致或略缩小）。
- **可选**：一行状态文字（专注中/已暂停/准备就绪）。
- **必须**：一个「还原」按钮或明确可点击区域，用于恢复主界面。
- **必须**：整窗（或除按钮外区域）可拖动。

## 配置与持久化（可选）

- 是否记住「上次是否以最小化模式关闭」：可选实现；若不做，每次启动均为完整主窗口。

## 验收标准

1. 主界面有「最小化」入口，点击后主窗口隐藏，出现仅显示时间（及状态）的小窗口。
2. 小窗口可被拖动到屏幕任意位置，时间在专注时持续更新。
3. 小窗口有「还原」入口，点击后小窗口关闭、主窗口恢复，状态与进入前一致。
4. 在最小化状态下计时结束，仍会弹出强制中断输入框，行为与现有一致。
