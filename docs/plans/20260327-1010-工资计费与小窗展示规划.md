## 原文档说明

本文档内容与「工资计费与小窗展示」产品/技术规划一致，便于在仓库内检索与评审。

## 目标

在主窗口和右下角小窗中新增「工资信息」展示，帮助用户在主窗口隐藏时，仍能实时看到：

- 距离发薪日还有几天
- 今日距离下班还有多久
- 按当天工资与工作时段，当前已经赚到多少钱
- 到达下班时间后，在界面内显示一条祝贺文字提示

## 已确认规则

- 发薪日规则：用户可设置「每月几号发」
- 工资计算口径：按设置的上班时间到下班时间，随着当天工作时段流逝自动累计工资
- 下班提示：不弹窗，不打断用户；只在界面中显示文字，例如：`下班时间到了，今天已经挣到 ￥XX.XX，恭喜你`
- 展示位置：主窗口可显示，但右下角小窗必须也显示，因为主窗口通常会被隐藏

## V1 范围

实现主窗口展示、右下角小窗展示、设置保存，不改记录页。

涉及文件：

- `src/renderer/index.html`
- `src/renderer/mini.html`
- `src/renderer/styles/main.css`
- `src/renderer/scripts/main.js`
- `src/main/main.js`

另：共用计算逻辑位于 `src/renderer/scripts/salary.js`，小窗脚本为 `src/renderer/scripts/mini-renderer.js`。

## 展示内容

### 主窗口

新增「工资信息」区域，建议包含：

- `发薪倒计时：还有 X 天发工资`
- `工作状态：距离下班 HH:MM:SS` 或 `距离上班 HH:MM:SS`
- `今日已赚：￥XX.XX / ￥日工资`
- 到达下班时间后，将工作状态改为：`下班时间到了，今天已经挣到 ￥XX.XX，恭喜你`

### 小窗口

由于空间有限，小窗建议展示精简版，但保留核心信息：

- `发薪：还有 X 天`
- `下班：HH:MM:SS` 或 `上班：HH:MM:SS`
- `已赚：￥XX.XX`
- 下班后改为短提示：`已下班，今日 ￥XX.XX`

未配置完整时，两处都显示简短引导文案，例如：`请先设置工资信息`。

## 设置项

在现有设置弹窗中新增以下字段：

- `上班时间`，例如 `09:00`
- `下班时间`，例如 `18:00`
- `日工资`，例如 `300`
- `发薪日`，例如 `15`

继续复用现有 `config` 存储，新增字段：

- `workStartTime`
- `workEndTime`
- `dailySalary`
- `payday`

## 计算规则

### 发薪倒计时

- 若今天未到发薪日，显示到本月该日还剩几天
- 若今天已过发薪日，显示到下个月发薪日还剩几天
- 若某月没有该日期，V1 按该月最后一天处理

### 上下班状态

- 当前时间早于上班时间：显示 `距离上班 HH:MM:SS`
- 当前时间位于上班到下班之间：显示 `距离下班 HH:MM:SS`
- 当前时间到达或超过下班时间：不弹窗，仅在主窗口和小窗中显示祝贺文字提示

### 今日已赚工资

- 公式：`已赚金额 = 日工资 * 已工作秒数 / 工作总秒数`
- 上班前：显示 `￥0.00`
- 上班中：按秒实时增长，保留两位小数
- 下班后：固定显示 `￥日工资`

## 小窗设计约束

- 小窗空间很小，需避免新增内容后出现滚动条或遮挡按钮
- 优先采用「短文案 + 单行信息」方案，必要时可适度增加小窗高度
- 小窗中的工资信息应始终低干扰，不影响开始、暂停、停止、记录、还原等核心按钮操作

## 实现思路

1. 在设置弹窗新增工作时间与工资配置项
2. 主进程扩展 `get-config` / `save-config`，支持新字段
3. 主窗口新增工资信息区域和样式
4. 小窗口新增精简版工资信息区域，并根据空间优化排版
5. 渲染进程在加载配置后启动每秒刷新逻辑
6. 主窗口与小窗共用同一套时间/工资计算逻辑，分别渲染成完整文案和精简文案
7. 到达下班时间后，仅更新界面中的状态文案，不触发弹窗、通知或其他打断行为

## 兼容与约束

- V1 仅支持同一天内上下班，不支持跨天班次
- V1 仅支持单段工作时间，不处理午休拆段
- V1 仅按日工资线性折算，不处理加班、扣税、奖金
- 未配置完整时不报错，仅显示提示

## 验收标准

1. 用户可在设置中配置上班时间、下班时间、日工资、发薪日
2. 主界面能显示距离发薪日还有几天、工作状态与今日已赚金额
3. 小窗也能显示发薪倒计时、工作状态与今日已赚金额
4. 到达下班时间后，主界面和小窗都显示下班后的文字提示
5. 全程不出现额外弹窗
6. 小窗新增信息后不出现滚动条，不遮挡现有按钮
7. 配置保存后，重启应用仍能恢复
