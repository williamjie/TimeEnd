# 专注/休息记录列表与按日/周/月查询 - 需求规划

## 需求概述

1. **列表展示**：在主界面提供一处「记录」入口，支持查看：
   - **专注记录**：每次专注的「开始时间 → 结束时间」、以及**专注结束时的输入内容**（即强制中断弹窗里填的内容）。
   - **休息记录**：每次休息的「开始休息 → 结束休息」的时间段。

2. **按时间排序**：列表内按时间排序（建议**结束时间倒序**，即最新的在前）。

3. **查询维度**：支持三种查询方式：
   - **按天**：查看某一天的专注与休息记录；
   - **按周**：查看某一周的记录及该周统计；
   - **按月**：查看某一月的记录及该月统计。

---

## 一、数据模型（需要持久化）

### 1. 专注记录（Focus Session）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识（如 UUID 或 时间戳+随机） |
| startTime | number | 开始时间戳（用户点击「开始」） |
| endTime | number | 结束时间戳（计时结束或用户点击「停止」） |
| durationMinutes | number | 专注时长（分钟），可由 start/end 计算 |
| inputContent | string | 专注结束时的输入内容（仅计时自然结束时有；中途「停止」可为空） |

- **何时写入**：  
  - 用户点击「开始」时记下 startTime（可先写一条「进行中」或等结束时再写）；  
  - **计时自然结束**：在用户提交中断弹窗后，写入一条完整记录（startTime, endTime, durationMinutes, inputContent）；  
  - **用户点击「停止」**：在停止时写入一条记录（startTime, endTime, durationMinutes, inputContent 为空）。

- 为简化，也可采用「结束时一次性写入」：在「自然结束并提交」或「点击停止」时，用当前内存中的 startTime 与 endTime 写一条记录。

### 2. 休息记录（Rest Session）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| startTime | number | 开始休息时间戳（专注结束或点击「停止」） |
| endTime | number | 结束休息时间戳（用户点击「开始」专注） |
| durationMinutes | number | 休息时长（分钟），可由 start/end 计算 |

- **何时写入**：  
  - 用户点击「开始」专注时，用当前的 `restStartTime` 与当前时间写一条休息记录（startTime = restStartTime, endTime = now），然后清零 restStartTime。

- **注意**：若应用在休息期间被关闭，该次休息可能没有 endTime；可存为「未结束」或约定 endTime 为 null，列表展示时单独说明（如「进行中」或按关闭时间近似）。

---

## 二、列表展示内容

- **每条专注记录**：开始时间、结束时间、专注时长（分钟）、结束时的输入内容（若有）。
- **每条休息记录**：开始时间、结束时间、休息时长（分钟）。
- **排序**：按**结束时间倒序**（最新在前）；同一天/周/月内也按此排序。

---

## 三、查询方式与统计

### 1. 按天查询

- **含义**：选择某一天（如「今天」或日历选某日），展示该日所有专注记录 + 该日所有休息记录。
- **界面**：日期选择（今天 / 昨天 / 或日期控件），列表展示当日记录，按结束时间倒序。
- **可选**：当日汇总（当日专注次数、当日专注总分钟、当日休息总分钟）。

### 2. 按周查询

- **含义**：选择某一周（如「本周」或按周选择），展示该周内所有专注记录与休息记录。
- **界面**：周选择（本周 / 上周 / 或周选择器），列表展示该周记录，按结束时间倒序。
- **统计**：该周「专注次数」「专注总分钟」「休息总分钟」等汇总。

### 3. 按月查询

- **含义**：选择某一月，展示该月内所有专注与休息记录。
- **界面**：月选择（本月 / 上月 / 或月份选择器），列表展示该月记录，按结束时间倒序。
- **统计**：该月「专注次数」「专注总分钟」「休息总分钟」等汇总。

---

## 四、界面与交互（建议）

1. **入口**：在主界面统计区附近增加「记录」或「历史」按钮，点击后打开**记录列表视图**（新页面、抽屉或弹窗，二选一即可）。
2. **视图内**：
   - 顶部：**查询维度切换**：「按天」/「按周」/「按月」。
   - 根据维度显示**时间选择**：按天 → 选日期；按周 → 选周；按月 → 选月。
   - 下方：**汇总区**（当前维度下的次数、专注总长、休息总长）。
   - 再下方：**列表**，每条为一行（专注：开始～结束 + 输入内容摘要；休息：开始～结束），按结束时间倒序。
3. **默认**：打开记录视图时，默认「按天」且「今天」，便于快速看今日记录。

---

## 五、技术要点（简要）

- **存储**：使用现有 electron-store，新增 key，如 `focusSessions: []`、`restSessions: []`，每条为上述结构。可考虑按 id 去重、按 endTime 排序存储或查询时排序。
- **主进程**：在「开始」「停止」「计时结束并提交」处打点，写入专注/休息记录；提供 IPC 如 `get-sessions`，参数为 `{ type: 'day'|'week'|'month', value: dateString | weekKey | monthKey }`，返回对应时间段内的 sessions + 汇总。
- **渲染进程**：记录页/弹窗内请求数据、渲染列表与汇总；时间选择可用原生 `<input type="date">` 或简单前后切换（今天/昨天/本周/上周等）。

---

## 六、验收标准（建议）

1. 主界面有「记录」入口，点击后能打开记录视图。
2. 记录视图支持按天/周/月切换，并可按选择展示对应时间段内的专注记录与休息记录。
3. 专注记录展示：开始时间、结束时间、专注时长、结束时的输入内容（若有）。
4. 休息记录展示：开始时间、结束时间、休息时长。
5. 列表按结束时间倒序排列。
6. 按周、按月视图下展示该周/该月的汇总（如专注次数、专注总分钟、休息总分钟）。
7. 新完成一次专注（自然结束并提交或点击停止）后，列表中能出现对应专注记录；新开始一次专注（从休息状态点击「开始」）后，列表中能出现对应休息记录。
