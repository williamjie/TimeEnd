# 最小化窗口增加开始/暂停/停止 - 需求规划

## 需求描述

在「最小化」小窗口（仅显示时间 + 状态 + 还原）上，增加**开始、暂停、停止**三个按钮，使用户可以在小界面内直接操作计时器，无需还原到主窗口即可控制专注的启停与暂停。

--

## 目标效果

| 项目 | 说明 |
|------|------|
| **新增控件** | 小窗口内增加「开始」「暂停」「停止」按钮，与主界面逻辑一致。 |
| **开始** | 点击后按当前配置的专注时长开始计时（与主界面「开始」一致）。 |
| **暂停** | 专注中时显示「暂停」，点击后计时暂停；已暂停时显示「继续」，点击后继续计时。 |
| **停止** | 专注中或已暂停时显示「停止」，点击后结束本次专注并进入休息（与主界面「停止」一致）。 |
| **按钮状态** | 未开始时：开始可用，暂停/停止禁用；专注中：开始禁用，暂停/停止可用；已暂停：开始禁用，暂停显示「继续」，停止可用。 |
| **本次休息** | 停止后（或专注自然结束后）进入休息状态时，小界面需**显示本次休息时间**（如「本次休息：X 分钟」），并随秒数增加；再次点击「开始」后休息清零，该显示隐藏。与主界面「本次休息」行为一致。 |

---

## 行为约定

1. **开始**
   - 小窗口加载后需能获取「专注时长」配置（与主界面设置一致）。
   - 点击「开始」后，主进程按当前专注时长启动计时，小窗口收到 `timer-update` 与状态更新，时间开始倒数。

2. **暂停 / 继续**
   - 专注中点击「暂停」→ 发送与主界面相同的暂停指令，计时暂停，按钮文案变为「继续」。
   - 已暂停时点击「继续」→ 发送恢复指令，计时继续，按钮文案变为「暂停」。

3. **停止**
   - 点击「停止」→ 发送与主界面相同的停止指令，计时结束，进入休息；小窗口状态变为「准备就绪」，时间显示恢复为配置的专注时长（如 25:00）。
   - **休息状态下**：小窗口需显示「本次休息：X 分钟」，并每秒更新；与主界面一致。再次点击「开始」后休息清零，该显示隐藏。

4. **与主窗口一致**
   - 小窗口与主窗口共用同一套计时逻辑（主进程 Timer）；在小窗口操作开始/暂停/停止后，若用户点击「还原」，主窗口显示的状态、剩余时间、是否在休息等应与小窗口一致。

5. **强制中断**
   - 在小窗口内计时自然结束时，仍按现有逻辑弹出强制中断输入框（父窗口为小窗口），行为不变。

---

## 技术要点

- **IPC**：小窗口通过现有 IPC 与主进程通信：`start-timer`（需传入专注时长）、`pause-timer`、`resume-timer`、`stop-timer`。主进程已处理上述事件，无需改主进程业务逻辑，只需小窗口在点击时发送对应 IPC。
- **配置**：小窗口在加载时向主进程请求一次配置（如 `get-config`），获取 `focusDuration`，用于「开始」时传入；或由主进程在创建小窗口时通过 `webContents.send` 下发当前专注时长。
- **状态同步**：小窗口已通过 `timer-update`、`mini-status` 接收计时与状态，据此控制按钮的可用/禁用及「暂停」与「继续」的切换即可。
- **本次休息**：主进程在进入休息时已发送 `rest-started(restStartTime)`，在再次开始专注时发送 `rest-cleared`。小窗口需监听这两类 IPC：收到 `rest-started` 后显示「本次休息」并本地按秒递增分钟数；收到 `rest-cleared` 后隐藏并清零。小窗口加载时若已在休息，可请求 `get-rest-state`（或主进程在 mini 创建时下发）以恢复显示。
- **布局**：小窗口适当增高或调整布局，容纳时间、状态、**本次休息（休息时显示）**、三个操作按钮及「还原」。按钮区域使用 `-webkit-app-region: no-drag`，避免拖动与点击冲突。

---

## 界面建议

- 自上而下：时间数字 → 状态文字 → **本次休息：X 分钟**（仅休息时显示）→ **开始 | 暂停 | 停止**（一行或两行）→ 「还原」。
- 按钮尺寸与字体可略小，保持小窗整体紧凑；风格与主界面按钮一致（如主色、圆角）。「本次休息」可用绿色或与主界面一致样式区分。

---

## 验收标准

1. 小窗口内可见「开始」「暂停」「停止」「还原」，且可拖动小窗口。
2. 点击「开始」后计时开始，时间倒数，状态为「专注中」；点击「暂停」后计时暂停，按钮变为「继续」；点击「继续」后计时继续；点击「停止」后计时结束，状态为「准备就绪」，时间恢复为配置时长。
3. 停止后（或专注自然结束后）小窗口显示「本次休息：X 分钟」并随秒数增加；再次点击「开始」后该显示消失。
4. 在小窗口内完成上述操作后点击「还原」，主窗口显示的剩余时间、状态、本次休息与操作结果一致。
5. 小窗口内计时自然结束时，仍会弹出强制中断输入框，行为与在主窗口时一致。
