# 快速使用说明

## 问题：Cursor 终端无法识别 npm

## 解决方案（按推荐顺序）

### 方案1：使用自动查找脚本（最简单）⭐

在 Cursor 终端中运行：

```powershell
.\find-nodejs.ps1
```

或者：

```powershell
.\fix-path.ps1
```

这两个脚本会**自动查找** Node.js 安装位置，您不需要知道路径！

### 方案2：使用批处理文件

双击运行 `快速修复.bat` 文件

### 方案3：手动设置（如果脚本找不到）

如果脚本找不到 Node.js，您可以：

1. **打开 Windows PowerShell**（不是 Cursor 的终端）
2. 运行：
   ```powershell
   where.exe node
   ```
3. 复制显示的路径（例如：`C:\Program Files\nodejs\node.exe`）
4. 在 Cursor 终端中运行：
   ```powershell
   # 替换为您的实际路径
   $env:PATH = "C:\Program Files\nodejs;$env:PATH"
   ```

## 修复后

修复成功后，运行：

```powershell
npm install
npm start
```

## 如果 Node.js 未安装

如果所有脚本都找不到 Node.js，说明可能未安装：

1. 访问 https://nodejs.org/
2. 下载并安装 LTS 版本
3. 安装时**务必勾选** "Add to PATH"
4. 安装完成后重启 Cursor
5. 再次运行 `.\find-nodejs.ps1`

## 推荐操作

**最简单的方式**：直接运行

```powershell
.\find-nodejs.ps1
```

这个脚本会尝试多种方法自动查找 Node.js，包括：
- 检查系统 PATH
- 检查常见安装路径
- 检查注册表
- 搜索 Program Files

您不需要知道 Node.js 安装在哪里！
