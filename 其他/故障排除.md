# 故障排除指南

## ❌ 错误：npm 命令未找到

### 错误信息
```
npm : 无法将"npm"项识别为 cmdlet、函数、脚本文件或可运行程序的名称。
```

### 原因
系统未安装 Node.js，或 Node.js 未添加到 PATH 环境变量。

### 解决方案

#### 方案1：安装 Node.js（推荐）

1. **下载 Node.js**
   - 访问：https://nodejs.org/
   - 下载 LTS（长期支持）版本
   - 推荐版本：v18.x 或 v20.x

2. **安装 Node.js**
   - 运行下载的安装程序
   - **重要**：安装时务必勾选 ✅ "Add to PATH" 选项
   - 按照向导完成安装

3. **验证安装**
   - 关闭当前终端窗口
   - 重新打开 PowerShell 或命令提示符
   - 运行以下命令验证：
     ```powershell
     node --version
     npm --version
     ```
   - 如果显示版本号，说明安装成功

4. **重新运行项目**
   ```powershell
   cd e:\company_code\TimeEnd
   npm install
   npm start
   ```

#### 方案2：使用 Chocolatey（如果已安装）

```powershell
choco install nodejs-lts
```

#### 方案3：手动添加到 PATH

如果 Node.js 已安装但未添加到 PATH：

1. 找到 Node.js 安装目录（通常在 `C:\Program Files\nodejs\`）
2. 复制该路径
3. 添加到系统环境变量：
   - 右键"此电脑" → 属性 → 高级系统设置
   - 环境变量 → 系统变量 → Path → 编辑
   - 新建 → 粘贴 Node.js 路径 → 确定
4. 重启计算机

## ❌ 错误：依赖安装失败

### 解决方案

1. **使用国内镜像源**
   ```powershell
   npm install --registry=https://registry.npmmirror.com
   ```

2. **清除缓存后重试**
   ```powershell
   npm cache clean --force
   npm install
   ```

3. **删除 node_modules 后重新安装**
   ```powershell
   Remove-Item -Recurse -Force node_modules
   npm install
   ```

## ❌ 错误：Electron 下载失败

### 解决方案

1. **设置 Electron 镜像**
   ```powershell
   npm config set electron_mirror https://npmmirror.com/mirrors/electron/
   npm install
   ```

2. **使用代理**
   - 配置系统代理或 VPN
   - 或使用 npm 代理设置：
     ```powershell
     npm config set proxy http://proxy-server:port
     npm config set https-proxy http://proxy-server:port
     ```

## 📞 获取帮助

如果以上方案都无法解决问题，请检查：

1. ✅ Node.js 是否正确安装
2. ✅ 是否在正确的项目目录
3. ✅ 网络连接是否正常
4. ✅ 防火墙是否阻止了下载
5. ✅ 磁盘空间是否充足
