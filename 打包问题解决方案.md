# Electron Builder 打包错误解决方案

## 错误原因

错误信息：`ERROR: Cannot create symbolic link : 客户端没有所需的特权`

**原因**：
- electron-builder 在解压 `winCodeSign` 工具时尝试创建符号链接
- Windows 创建符号链接需要管理员权限
- 当前用户没有足够的权限

## 解决方案

### 方案1：禁用代码签名（推荐，已配置）⭐

对于个人项目或内部使用，不需要代码签名。已在 `package.json` 中配置：

```json
"win": {
  "sign": false
}
```

现在可以直接打包：

```powershell
npm run build
```

### 方案2：以管理员身份运行

如果需要代码签名，可以：

1. **右键点击 PowerShell**，选择"以管理员身份运行"
2. 进入项目目录：
   ```powershell
   cd e:\company_code\TimeEnd
   ```
3. 运行打包命令：
   ```powershell
   npm run build
   ```

### 方案3：启用 Windows 开发者模式

启用后，普通用户也可以创建符号链接：

1. 打开"设置" → "更新和安全" → "开发者选项"
2. 启用"开发人员模式"
3. 重启计算机
4. 重新运行打包命令

### 方案4：手动清理缓存

如果缓存损坏，可以清理后重试：

```powershell
# 删除 electron-builder 缓存
Remove-Item -Recurse -Force "$env:LOCALAPPDATA\electron-builder\Cache" -ErrorAction SilentlyContinue

# 重新打包
npm run build
```

## 当前配置

已更新 `package.json`，禁用了代码签名：

```json
"win": {
  "target": "nsis",
  "icon": "assets/icon.ico",
  "sign": false  // 已添加：禁用代码签名
}
```

## 现在可以打包

直接运行：

```powershell
npm run build
```

打包后的安装程序将位于 `dist` 目录。

## 注意事项

- **禁用代码签名**：安装时 Windows 可能会显示"未知发布者"警告，这是正常的
- **个人使用**：对于个人项目，不需要代码签名
- **商业发布**：如果需要商业发布，需要购买代码签名证书，然后移除 `"sign": false`
